                        # 简介

                        # 背景

                        • 减少重复代码逻辑，将工作集中于业务逻辑上
                        • 开发快捷高效，将时间用于业务代码上
                        • 经常用到弹窗，展示数据、新增、编辑等业务逻辑；

                        # 特性

                        • 灵活度高: 除默认提供通用基础表单类型，给开发者提供了更灵活的自定义的写法，所有代码逻辑均可配置

                        # 使用

                        • js中引入组件 import ComDialog from "/@/components/ComDialog";
                        • html中注入组件 <ComDialog />

                        # 属性

                        # ComDialog属性

                        • ComDialogelement plus Dialogopen in new window中所有的属性
                        • 以下所列是基于其原有属性的扩展

                        # 属性

                        属性名默认值类型描述用法
                        title“ ”StringDialog弹窗顶部的标题title="修改菜单权限角色"
                        showFullscreentrueBoolean是否显示全屏按钮,默认显示:showFullscreen="true"
                        draggabletrueBoolean是否可以拖拽, 默认可拖拽:draggable="true"
                        showFootertrueBoolean是否展示底部按钮:showFooter="true"
                        dialogWidth"50%"String弹框宽度dialogWidth="700px"
                        labelWidth"150"String标签的长度labelWidth="100"
                        className“ ”String内容区form表单的自定义样式选值：1、默认每行一列；2、grid-2：每行2列；3、grid-3：每行3列；
                        flex"1"String内容区form表单的自定义样式1 单列,2 双列,3 三列
                        data{}Objectform表单数据需和表单配置项config保持统一
                        config[]Arrayform表单配置项需和表单数据data保持统一

                        # 查看源代码

                        <template>
  <ComDialog
    v-model="visible"
    dialogWidth="600px"
    :title="dialogTitle"
    :data="dialogData"
    :config="data.dialogConfig"
    @submit="submit"
  />
</template>

<script setup lang="ts">
  import { ref, reactive, toRef } from "vue";
  import ComDialog from "/@/components/ComDialog";
  
  const visible = ref(false);
  const dialogTitle = ref("修改菜单权限角色");
  const data = reactive({
    dialogConfig: [
      {
        name: "业务系统:",
        type: "select",
        label: "system",
        labelKey: "value",
        listName: [
          {
            title: "卓保供",
            value: 1
          },{
            title: "卓集市",
            value: 2
          }
        ]
      },
      {
        name: "菜单权限名称:",
        type: "input",
        label: "menuName"
      },
      {
        name: "菜单权限URL:",
        type: "input",
        label: "url"
      },
      {
        name: "菜单权限排序:",
        type: "input",
        label: "sort"
      },
      {
        name: "菜单权限图标地址:",
        type: "input",
        label: "iconUrl"
      }
    ],
    dialogData: {},
  });
  const dialogData = toRef(data, "dialogData");

  const submit = val => {
    // val为组件内的form表单数据
    console.log("保存", val);
  };
</script>

                        # 插槽

                        插槽名说明
                        title对话框标题的内容；默认为父组件提供的title值。
                        content对话框内容区；默认为类型为select、checkbox、radio、input的form表单。
                        footer对话框底部按钮操作区的内容；默认为“取消”、“保存”按钮。

                        # 事件

                        事件名说明
                        submitDialog保存按钮的回调，返回表单数据。
                        changeDialog select选择框change的回调，返回当前item项。
                        closeDialog 关闭的回调。
